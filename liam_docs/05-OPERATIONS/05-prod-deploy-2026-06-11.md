## Drift Skema yang Ditemukan

Akar masalah: skema prod **tidak dibangun via migration file lokal**. History
`supabase_migrations.schema_migrations` di prod cuma 3 entri (`initial_schema`,
`generate_order_id_function`, `create_preregistrations`) sedangkan repo punya
9 file migration. Kemungkinan tabel awal dibikin manual / via dashboard, lalu
migration `create table if not exists` di-skip → klausa `on delete cascade` &
constraint baru tidak ikut terterapan.

### Bug Critical (BREAKING)

| # | Constraint / Kolom | Prod | Mestinya | Dampak |
|---|---|---|---|---|
| 1 | `profiles.id_fkey → auth.users` | NO ACTION | CASCADE | hapus user gagal total (error 23503) |
| 2 | `orders.id` default | (kosong) | `generate_order_id()` | insert order baru gagal NOT NULL |
| 3 | `orders.delivery_address` | NOT NULL | nullable | insert order tanpa alamat gagal |
| 4 | `orders.payment_method` check | `transfer_bank|qris` | `transfer_bank|qris|cod` | nilai 'cod' ditolak |
| 5 | `order_items.order_id` | nullable | NOT NULL | bisa orphan item |
| 6 | `order_items.category` | (tidak ada) | text nullable | snapshot kategori hilang |
| 7 | `order_items.created_at` | (tidak ada) | timestamptz default now() | tidak ada audit timestamp |

### Bug Non-Breaking (preserve behavior)

| # | Constraint | Prod | Mestinya | Dampak |
|---|---|---|---|---|
| 8 | `orders.user_id → profiles` | NO ACTION | SET NULL | preserve riwayat order saat user dihapus |
| 9 | `meal_entries.recipe_id → recipes` | NO ACTION | SET NULL | snapshot meal tetap aman |
| 10 | `subscriptions.user_id → profiles` | NO ACTION | CASCADE | drift dari niat awal |

### Hardening
| # | Item | Sebelum | Sesudah |
|---|---|---|---|
| 11 | `prevent_role_change()` ACL | EXECUTE granted ke anon/authenticated | revoke total dari role API |

### Drift yang TIDAK Disentuh (sengaja konservatif)

| Item | Status | Alasan |
|---|---|---|
| `subscriptions` table | tabel ada, tidak ada migration sebelumnya | dibungkus migration baru `if not exists` agar repo jadi sumber kebenaran tanpa ubah skema |
| `orders.service_fee` | kolom extra di prod | nullable & tidak dipakai frontend, harmless |
| `orders.payment_status` | kolom extra di prod | sama, default `'pending'`, tidak dipakai code |
| `orders.order_status` | kolom extra di prod | sama, default `'received'`, tidak dipakai code |
| `profiles.gender` | kolom extra di prod | tidak ada di migration mana pun, tidak dipakai code |
| Policy duplikat di `weekly_plans` (`plans_owner` + `weekly_plans_owner`) | dibiarkan | efek sama, kosmetik |
| Policy duplikat di `recipes` & `recipe_ingredients` | dibiarkan | efek sama, kosmetik |
| Migration history mismatch (3 vs 9) | dibiarkan | repo punya orang lain, paksa sync = risiko |
